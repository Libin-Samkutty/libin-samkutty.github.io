Libin Samkutty

AI Quality Engineer • Jeavio India Pvt. Ltd.

AI Quality Engineer building evaluation infrastructure for AI systems: LLM evaluation, RAG pipeline testing, and production monitoring using DeepEval, RAGAS, and custom instrumentation.

I started in QA directly out of college because testing genuinely interested me. The discipline of understanding a system well enough to find where it breaks, and the challenge of making that process repeatable and fast, drew me in from my first project as an intern in January 2022.

Starting out at Jeavio, the work involved Selenium UI automation, JMeter load testing, and OWASP ZAP security scans, alongside manual regression cycles with a small team and no automation baseline.

Over 4+ years, built QA capability from scratch on a live production platform. Moved from writing scripts to owning QA strategy for a conversational AI health application serving users across India, Kenya, and Nigeria. That meant building Botium regression suites, cloud health-check pipelines, AI response classification test suites, and API automation that runs on every release cycle, spanning web, REST API, conversational AI, cloud infrastructure, and database layers.

Right now, that means running production AI evaluation frameworks using DeepEval and RAGAS: golden datasets, multi-dimensional eval scoring, trace instrumentation, and continuous monitoring for hallucination, faithfulness, and retrieval quality across a live RAG platform serving users across 6 countries. The shift from conversational flow automation to AI output evaluation changed the discipline: the system is non-deterministic, the failure modes aren't binary, and the eval framework itself needs to be versioned and maintained like any other production system.

Principles

How I think about QA

"AI systems don't have a correct answer to assert against; they have a distribution of acceptable outputs. That's not a reason to skip evaluation; it's a reason to build better evaluation infrastructure."

Applied this by building DeepEval and RAGAS pipelines with golden datasets, multi-dimensional scoring, and trace instrumentation, treating the eval framework itself as a production system that needs versioning and maintenance.

"Automation should reduce toil, not replace judgment."

Built 140+ automated conversational flows to eliminate repetitive regression cycles, while keeping manual judgment for AI response quality, edge case triage, and platform-specific rendering.
